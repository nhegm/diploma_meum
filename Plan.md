#### Ввод валидных данных в поля форм (4)

Под валидными данными для полей будут пониматься следующие значения:

      1. Поле "Номер карты": 16-тизначный номер без пробелов, состоящий только из цифр (например: 1600123499991111).
      2. Поле "Месяц": 2-значный номер месяца (например: 03, 11). Если введён текущий год, то месяц не может быть меньше текущего месяца.
      3. Поле "Год": 4-хзначный номер года, минимум равный текущему, но не более 10-ти лет от текущего (например: 2023, 2033).

      5. Поле "CVC/CVV": необходимо уточнить у разработчика, сколько цифр должно быть в данном коде.

### 2. Перечень используемых инструментов с обоснованием выбора.

1. Автоматизированное тестирование с использованием среды IDEA для языка Java.

2. Использование JUnit5 как основу для написания тестов.

3. Проект на базе gradle для отслеживания правильности работы тестов.

4. Использование Selenide для UI-тестирования приложения.

5. Использование Allure для создания репортов.

### 3. Перечень необходимых разрешений, данных и доступов.

      1. Необходимо получить формат Имени владельца карты
         1.1. Допускаются ли ещё какие-то символы, кроме латинских букв
         1.2. Допускается ли только одно имя (слово)
         1.3. Допускается ли более двух слов
         1.4. Допускаются ли имена с дефисами: Leonid Bugrov-Razuvaev
         1.5. Допускается ли написание типа: Roman L. (вместо фамилии первая буква фамилии и точка)
         1.6. Есть ли ограничение на длину имени
      2. Необходимо получить данные о длине кода CVC
         2.1 Длина в 3 символа или в 4 (зависит от типа платёжной системы)
      3. Необходимо получить Имя, Месяц, Год и CVC для карты под номером "4444 4444 4444 4441"
      4. Необходимо получить Имя, Месяц, Год и CVC для карты под номером "4444 4444 4444 4442"

### 4. Перечень и описание возможных рисков при автоматизации.

1. Риск остановки тестирования в случае, если не были получены данные о формате Имени и кода CVC.
2. Риск задержки тестирования в случае, если не были предоставлены данные о картах "4444 4444 4444 4441" и "4444 4444 4444 4442".
